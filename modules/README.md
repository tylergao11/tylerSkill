# Reusable Capability Modules

Reusable module specifications live here.

Use `docs/references/reusable-modules.md` before promoting project code into a
module. A module should not be created just because future reuse is imagined.
Promotion must be earned through repeated need and clean boundaries.

Each module should eventually include:

- Module Contract
- Small public API
- Example usage
- Tests
- Failure modes
- Portability notes
